How TradeRecon Works

The complete post-trade reconciliation workflow — from raw data to audit-ready resolution — in four clear phases.

1

Upload & Ingest Trade Data

Your operations team uploads internal trade blotters via the Upload Trades page. Accepted formats include CSV (from Bloomberg, Murex, Calypso) and JSON. Alternatively, individual trades can be submitted via the manual entry form for OTC or bespoke instruments.

CSV / JSON / XML Manual Entry Form Schema Validation
2

Trigger the Reconciliation Engine

Navigate to the Reconciliation page and click Trigger Run. The engine loads your internal positions, fetches counterparty confirmations, and applies multi-rule matching logic. The 4-step loader shows real-time progress. Results are stored and available immediately upon completion.

3-Second Engine Run Multi-Asset Rules ISIN / CUSIP Matching
3

Review & Resolve Exceptions

The Exceptions page surfaces all breaks with severity ratings (Critical → Low), exception type, counterparty, variance amount, and assigned owner. Analysts can update status, add comment threads, and escalate items. Every action is logged.

Critical / High / Medium / Low 7 Exception Types Comment Threads
4

Export Reports & Audit Trail

On the Reports page, export filtered datasets as CSV using the native Blob download API — no server required. The Audit Log provides an immutable, time-stamped record of every user action for MiFID II, EMIR, and SOC-2 compliance.

CSV Export (Blob API) Full Audit Trail MiFID II Ready
